How It Works

Up and Running in 5 Minutes

Three simple steps from download to accepting live payments on your WooCommerce store.

1
Download the Plugin

Grab the latest release from GitHub. It's a standard WordPress plugin zip — no build steps, no dependencies.

2
Upload & Activate in WordPress

Go to Plugins → Add New → Upload Plugin, select the zip, activate — and PaymentHood appears in your WooCommerce payment gateways list.

3
Connect Your PaymentHood Account

Paste your PaymentHood API key in the plugin settings, choose sandbox or live mode, and save. Your store is now ready to accept payments.

FAQ

Frequently Asked Questions

Which WooCommerce versions does the plugin support?
PaymentHood for WooCommerce requires WooCommerce 6.0 or higher on WordPress 5.8+. It is tested with the latest stable WooCommerce releases.
Does PaymentHood support WooCommerce Subscriptions?
Not at this time. The current version supports one-time checkout payments only. Subscription / recurring billing support is planned for a future release.
How do I get a PaymentHood API key?
Create a free PaymentHood account, add your chosen payment provider in the dashboard, and generate an API key. Then paste that key into the WooCommerce plugin settings page.
Can I test payments before going live?
Yes. The plugin includes a sandbox mode toggle. When enabled, all transactions are routed through your payment provider's sandbox environment so you can verify the full checkout flow without processing real payments.
Does the plugin work with the WooCommerce Block Checkout?
Yes. PaymentHood registers as a standard WooCommerce payment gateway and is compatible with both the Classic Checkout and the newer Block-based Checkout experience.
Is there a cost to use the plugin?
No. The PaymentHood WooCommerce plugin is free to download and use. You pay only the transaction fees charged by your chosen payment provider — there are no additional fees from PaymentHood.
Can I switch payment providers without reinstalling the plugin?
Yes. Provider switching happens in your PaymentHood dashboard — just activate a different provider there. Your WooCommerce plugin continues working without any changes on the WordPress side.
