FAQ

Frequently Asked Questions

Which Joomla and VirtueMart versions does the plugin support?
PaymentHood for VirtueMart runs on Joomla 4.0 or higher with a supported VirtueMart 4 release. It is tested against the latest stable Joomla and VirtueMart versions.
Does PaymentHood support subscriptions or recurring billing?
Not at this time. The current version supports one-time checkout payments only. Subscription / recurring billing support is planned for a future release.
How do I get a PaymentHood API key?
Create a free PaymentHood account, add your chosen payment provider in the dashboard, and generate an API key. Then paste that key into the VirtueMart payment method settings page.
Can I test payments before going live?
Yes. The plugin includes a sandbox mode toggle. When enabled, all transactions are routed through your payment provider's sandbox environment so you can verify the full checkout flow without processing real payments.
How is the plugin installed on Joomla?
It installs like any Joomla extension. Download the package, go to Extensions → Manage → Install, upload the file, then enable the PaymentHood payment plugin. It then appears as a payment method inside VirtueMart.
Is there a cost to use the plugin?
No. The PaymentHood VirtueMart plugin is free to download and use. You pay only the transaction fees charged by your chosen payment provider — there are no additional fees from PaymentHood.
Can I switch payment providers without reinstalling the plugin?
Yes. Provider switching happens in your PaymentHood dashboard — just activate a different provider there. Your VirtueMart plugin continues working without any changes on the Joomla side.
